## 📌 Описание проекта
**E-Com Churn Radar** — это end-to-end Data Science проект, направленный на прогнозирование оттока клиентов (Customer Churn) в сфере электронной коммерции. Удержание текущих клиентов обходится бизнесу значительно дешевле привлечения новых, поэтому раннее выявление пользователей из группы риска — критически важная задача для e-commerce платформ.

В данном проекте реализован полный цикл разработки: от разведочного анализа данных (EDA) и борьбы с дисбалансом классов до обучения моделей машинного обучения и выгрузки предиктивных инсайтов в интерактивный дашборд Power BI.

## ⚙️ Этапы реализации проекта

### 1. Исследовательский анализ данных (EDA) & Предобработка
* Анализ структуры данных, заполнение пропущенных значений и обработка аномалий.
* Изучение распределения ключевых признаков (активность пользователей, частота покупок, средний чек, количество жалоб, использование промокодов).
* Кодирование категориальных переменных и масштабирование числовых признаков для корректной работы ML-алгоритмов.

### 2. Машинное обучение & Прогнозирование оттока
* **Дисбаланс классов:** Устранена проблема сильного дисбаланса между оттекшими и лояльными клиентами (применены методы балансировки/взвешивания классов).
* **Моделирование:** Проведено обучение и валидация ансамблевых моделей классификации (Random Forest, Gradient Boosting).
* **Оценка качества:** Модели оптимизировались с упором на метрику **Recall** (чтобы минимизировать пропуск уходящих клиентов) и **ROC-AUC** для оценки общей разделяющей способности модели.

### 3. Экспорт данных для бизнес-анализа
Для того чтобы результаты ML-модели приносили реальную пользу бизнесу, код генерирует две финальные витрины данных:
1. `churn_results_for_BI.csv` — скоринг клиентской базы (фактические данные + предсказанная вероятность оттока по каждому пользователю).
2. `feature_importance.csv` — веса и значимость признаков, определяющие, какие факторы сильнее всего влияют на уход клиентов.

## 📊 Интерактивный дашборд Power BI
Выгруженные файлы интегрированы в Power BI для создания аналитической панели управления. 

**Ключевой функционал дашборда:**
* **Churn Risk Monitoring:** Сегментация клиентов по уровням риска ухода (Низкий, Средний, Высокий) на основе вероятностей из ML-модели.
* **Feature Drivers:** Визуализация топ-факторов, стимулирующих отток (например, количество дней с последней покупки, падение активности, негативные отзывы). Это позволяет маркетологам понимать *почему* уходят клиенты.
* **Targeted Campaigns:** Фильтрация списков пользователей с высоким риском оттока по категориям товаров и регионам для запуска точечных push-уведомлений или промо-рассылок.
